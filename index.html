¿CÓMO INICIO?

La traducción inicio el 1 de mayo del 2020.Se partió desde el tutorial que existía en la Página de Python Argentina para facilitar la traducción se migro archivos .po ya que estos permiten manejar distintos idiomas para su traducción. Hace aproximadamente cuatro años la documentacion fue acepta en el repositorio oficial de Python


Link del Repositorio
https://github.com/python/python-docs-es
Link de la documentación oficial
https://docs.python.org/es/3/

¿CÓMO COLABORO?

  • Se hace un fork del repositorio https://github.com/python/python-docs-es
  • Busco un issue abierto pido que se me asigne
  • Envió la traducción y creo un pull request
  • Luego un colaborador hará un review

FORK

        
          #Clona el fork del repositorio que acabas de crear:

          git clone git@github.com:usuariogithub/python-docs-es.git
          
          #Ingresa en la carpeta que git clone creó en tu computadora:
          
          cd python-docs-es/
          
          #Agrega el repositorio original como «upstream»:
          
          git remote add upstream https://github.com/python/python-docs-es.git
          
          #(Opcional) Crea un entorno virtual y actívalo:
          
          python -m venv env
          source env/bin/activate   # macOS y Linux
          env\Scripts\activate.bat  # Windows
          
          #(Opcional) Instala los requerimientos del proyecto:
          
          pip install -r requirements.txt

ISSUES

Tenemos una lista de issues en GitHub en dónde vamos coordinando el trabajo realizado para no traducir dos veces lo mismo. El proceso para traducir un archivo es el siguiente:

  • Elige cualquiera de los issues que no están asignados a otra persona.
  • Deja un comentario en el issue diciendo que quieres trabajar en él.
  • Espera a que un administrador te asigne el issue.
  • ¡Empieza a traducir!

TRADUCCIÓN

Una breve guía de terminos que te pueden servir como referencia Memoria de Traducción

Tenemos una lista de issues en GitHub en dónde vamos coordinando el trabajo realizado para no traducir dos veces lo mismo. El proceso para traducir un archivo es el siguiente:

        
          

    #Selecciona un archivo para traducir

    #Verifica que estás en la rama principal del repositorio, 3.12

    git checkout 3.12

    #Crea una rama nueva en base al artículo en el que vayas a trabajar. 
    
    git checkout -b traduccion-glosario

    Empezamos a traducir###

    #Cuando hayas terminado tu sesión,lo envias a GitHub
    
    git add library/glosario.po
    git commit -m 'Traducido archivo library/glosario'
    git push origin traduccion-glosario
